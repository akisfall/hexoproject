打開Visual Studio Code

有一種方式是直接在電腦上面打開Vscode，這樣的開啟方法會無法流暢的指定製作的專案資料夾。 因此，提供另外一個方法，直接使用cmd（上一張提到的命令提示字元）開啟Vscode。 首先，如果你的專案資料夾想要放在磁碟槽的一個地方可以直接打上
cd （想打開的資料夾）
但是，如果你想要把cmd切換成任意磁碟槽的任何一個地方，就要把上述的指令改為
cd /d （任意磁碟槽）:/（想打開的資料夾）

虛擬環境

整理好你的開發環境

虛擬環境，Python內建的一個技術（於3.9版中內建）。 虛擬環境最主要是將每個專案需要的環境分開來，能夠將各個環境明確的分開，也不必占用全域環境（只裝在C槽的Python）的記憶體空間。 另外，也有可能當初開發的程式會需要某個套件（套件，將別人寫的程式碼庫直接來套用）的1.0版，也可能現在開發的程式會需要2.0版，但因為1.0版與2.0版的套件不可能一起存在， 所以虛擬環境便能夠解決這種尷尬的情況。

Python 應用程式通常會用到不在標準函式庫的套件和模組。應用程式有時候會需要某個特定版本的函式庫，因為這個應用程式可能需要某個特殊的臭蟲修正，或是這個應用程式是根據該函式庫特定版本的介面所撰寫。 這意味著不太可能安裝一套 Python 就可以滿足所有應用程式的要求。如果應用程式 A 需要一個特定的模組的 1.0 版，但另外一個應用程式 B 需要 2.0 版，那麼這整個需求不管安裝 1.0 或是 2.0 都會衝突，以致於應用程式無法使用。—節錄自 虛擬環境簡介

虛擬環境在一般的教學較少提到，不過虛擬環境能夠整合開發環境，才會在這裡提及。

建立虛擬環境

首先，建立虛擬環境需要開啟在Windows內的cmd（命令提示字元）。
如果是Windows 10，只要在鍵盤上按下 Windows鍵（Windows logo）＞直接打上cmd＞按下Enter 就可以進入cmd了。

最後只要輸入
python -m venv （你想要的名子）
以上，建立虛擬環境就完成了。
舉例來說，我想建立一個叫做PythonFun的虛擬環境。 只要在cmd打上
python -m venv PythonFun

打開虛擬環境

建立好虛擬環境後，會發現有個跟你先前輸入的名子同名的資料夾在同一個位置底下（假設輸入的名子是PythonFun）。
那建立好的虛擬環境該如何開啟呢？方法是把PythonFun裡面的Scripts\activate.bat檔打開來。
有兩種方法，一種是絕對路徑，另一個是相對路徑。
絕對路徑就是指很完整的路徑，從在C槽或D槽，一直到資料夾內的檔案路徑等，都很詳細的寫下來。
相對路徑就是不必撰寫得很完整的路徑，可以省略掉C槽或D槽，就可以直接開啟檔案。

絕對路徑：
D:\PythonFun\Scripts\activate
相對路徑比較麻煩，會需要你停留在D槽裡面。
直接輸入：
PythonFun\Scripts\activate
請注意是\（Enter上面的斜線）而非是/（shift旁邊的斜線！）

PIP

在進行更進一步的解釋前，我們先來談談pip。pip是一個管理各個套件的內建工具。提供了Python能夠查詢、刪除以及安裝各大程式碼庫的管理工具。
通常在書本上看到的套件安裝法為：
pip install 套件名稱

以上的方法是在全域的環境直接安裝，那如果我們要在虛擬環境中安裝的話要怎麼辦？ 很簡單，先用前一Part的方式打開虛擬環境後，輸入同上的指令
pip install 套件名稱

如果想要查詢安裝了什麼套件，可以直接輸入
pip list

只要在虛擬環境中安裝的pip套件，跟外面的Python是完全不可相通的喔！
如果你想要關閉虛擬環境的話可以使用
PythonFun\Scripts\deactivate來關閉喔！！

Requirements.txt

Requirements.txt，如果玩過電腦遊戲一定知道，有個區塊一定會有最低配置。 最低配置就是你可以最少最少把遊戲打開的標準，也就是執行遊戲的「必要」條件。 如果想要你的程式碼能夠被安全執行，就會需要產生一個requirments.txt來告訴大家執行你的程式會需要甚麼樣的套件來避免錯誤。
建立Requirements.txt只要輸入
pip freeze > requirements.txt
就可以建立Requirement.txt了。 只要你的朋友想要執行你的程式碼，只需要叫他們把cmd打開來後，打上
pip install -r requirements.txt
就完成基礎的Setup了。

補充一下，pip freeze可以產生你環境裡面的各個套件的版本，而下面指令的意思是
pip freeze > requirements.txt
我把pip freeze產生出來的結果匯入（>）到requirements.txt內。

原始碼編輯器介紹

原始碼編輯器（程式碼編輯器）是一種工具，提供工程師們在撰寫程式時能夠有增加效率的功能， 你可以從軟體上面看到五顏六色的各種語法標記（Syntax Highlighting）與其他常用的功能 例如：剪下複製貼上、快速縮排、重構程式碼等等，都必須要透過原始碼編輯器才可能做到。 這個世界上有很多不同的原始碼編輯器，在眾多的原始碼編輯器中，只需要一個挑選的原則。
自己喜歡或者習慣的就好！
基本上不必太糾結於其他人常用的編輯器，只要在乎自己習慣或喜歡的編輯器就好了。 接下來，我們會介紹幾個常見的編輯器。

Visual Studio Code

Visual Studio Code又稱為VScode，是由微軟開發，是一開放原始碼與免費的原始碼編輯器。 由於有著很簡潔又簡單的介面，與此同時也有很複雜的設定，使得其成為很有彈性的原始碼編輯器 變成新手好入門，而老手可以依照自己的需求去設定自己想要的介面，便躍升成為在軟體開發者間最常使用的編輯器。 除了彈性的設定外，還有著能夠支援大部分程式語言的語法標記的強大功能，以及社群對於其撰寫的附加元件（extension）都是Vscode的主力喔！

以下是 StackOverFlow（非常知名的程式論壇網站，只要一找關於程式的問題，一定知道它的存在。）所製作的2023年年度調查。

Pycharm

PyCharm作為JetBrains為了Python社群所開發的原始碼編輯器，其實介面設計與 InetlliJ （一個撰寫Java的原始碼編輯器）非常相似。 PyCharm有個很好的優點，除了他與Intellij有著一樣的介面，同時他也非常嘮叨。對於程式碼格式上的要求非常嚴格 。 除了對程式碼格式嚴格謹慎外，Pycharm還有個auto refactor（自動重構，俗稱一鍵排版）的功能。 但有著一個缺點，非常吃電腦資源，根據一篇SOF的問題所述PyCharm可以一次就吃到1.5G的RAM，造成可能沒辦法一次開太多實況掛台，跟聽YT音樂的情形。（假設你的電腦比較差點）

Spyder/Jupyter

Spyder大概是在學校裡面最常見的原始碼編輯器了，最主要的原因是Spyder能夠內建了許多的第三方套件（簡稱為套件，意旨由其他人所撰寫的程式碼） 除了能夠省略需要使用pip安裝的過程（後面會提），也是帶新手入門Modules的好選擇。

另外Jupyter跟Spyder都是屬於Anaconda旗下的原始碼編輯器，Jupyter特別的地方是網路編輯器。 也就是說，所有程式的撰寫與執行都是在網頁上一並執行喔！ 而且他可以建立很多個筆記，等於是將很多不同的程式碼放在一起，然後想要測試哪個區段就可以到哪邊測試喔！ 註：學過AI的朋友們，應該也知道Google Colab也是同樣的介面喔！

NeoVim

最後的最後，我想要來介紹這個特殊的原始碼編輯器。NeoVim。
這個編輯器有個特別的地方，他需要你全程「打字！」，你必須要透過背起許多指令以及各個按鍵的功能，才能夠使其有「程式碼編輯器」的功能！ 太多了嗎？還有呢，因為它內建的功能不足以成為一個合格的編輯器，所以你要透過安裝其他人編寫的插件才能夠讓他成為合格的程式碼編輯器。
好的，那…它為甚麼會出現在這個世上呢？既然那麼難用的話？
因為它很輕量簡潔、有彈性以及最重要的你不必動用你的滑鼠！ 由於寫程式的過程大部分都在透過鍵盤打字，然而，你去動用你的滑鼠會需要伸過去的時間、滑動的時間以及點擊的時間。 NeoVim或者Vim（NeoVim的爸爸）就是完全解決這個問題，它們只要用鍵盤就能做到跟Vscode一樣的功能！根據一篇 Medium的文章所述 ，Neovim成功地減少了許多開發的時間，因為減少了動用滑鼠的時間。
