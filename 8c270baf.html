第一步：找到財政部的發票兌獎網站

首先，可以直接透過這裡進入統一發票的兌獎網站。
找到序號為1或者2的標題，標題應該會是XXX（民國）年 0X~0X月的樣子，如圖

點進去之後就會看到特別獎特獎與頭獎的號碼了！

第二步：尋找標籤

網頁的基底主要是由HTML、CSS以及JavaScript甚至於PHP來組成的。
在這裡，我們不需要複雜的前端後端框架的知識，在這裡最好的情況是有API，可惜財政部並沒有針對發票兌獎製作API。

關於API

API是指Application programming interface。中譯應用程式介面
API主要是讓前端與後端能夠溝通的一種技術，它的工作是將一個使用者想要的資料，又稱之為
Request（直翻為要求）傳輸給後端的系統，可能是代理伺服器或者主伺服器。
Request傳輸出去後，系統（伺服器）收到後會去分析後了解使用者想要的資料，接著回傳一個對應的Response（回應），通常會是JSON格式。

JSON

Json是很常見的前後端溝通格式，其來源延伸自JavaScript中的Object語法。
因為其簡單又直接的溝通方式，以及延伸自JS的語法，便成為了網路中前端與後端溝通的格式。

網頁三巨頭

網頁三巨頭是指過去組成網頁架構的HTML以及CSS，以及現代較常使用的JavaScript組成。
HTML是由一個又一個的標籤(tags)組成，像是<div>Hello world<div/>，它組成了網頁的架構，也是最基層的存在。比喻來講，就是建築的鋼筋與水泥。
CSS是由選擇器(selector)與大括號內的語法組成的樣式設定表。它能夠設定從簡單的顏色字體大小，到直接決定整個網頁的排版與動畫，是前端語言重要的存在。比喻說，它就是建築的設計與材質等。
JavaScript是一個腳本語言，掌管著包含計算、後端的資料處理，本專案用不到，不細緻解釋。
如果有興趣可以到 Javascript info 來了解。

網頁檢測器

按下F12或者右鍵->檢測，就能開啟網頁檢測器。
網頁檢測器可以讓我們看網頁原始碼，有很多彩色文字以及英文的部分就是HTML。
只要按下左上角有一個滑鼠鼠標+正方形的圖示，然後將自己的滑鼠移動到中獎號碼上面，就可以找到中獎號碼的所在位置了。

以上，這一步就完成了，但先不要關閉掉網頁喔！

第三步：建置環境

（以下用Windows作為示例，Linux以及Mac請參照）
首先，可以建立一個資料夾，為這個專案取個名子吧！
然後建置一個虛擬環境給這個專案，如果不清楚的話可以參照
打開cmd，在你建立的資料夾底下輸入python -m venv (專案名稱)
例如：D:/crawlerReceipt。然後使用`(專案名稱)\Scripts\activate.bat打開虛擬環境。
我們需要的套件如下：

  • beautifulsoup4
  • requests

最後就輸入python -m pip install beautifulsoup4以及python -m pip install requests即可。

第四步：開始寫程式。

首先，我們可以回頭觀察財政部的發票兌獎網站，我們可以在URL發現一些有用的訊息。

URL是甚麼？

URL就是指網址，由三個部分組成，分別為協定、網域以及路徑組成。
用本網站來舉例 https://akisfall.github.io/hexoproject
https指的就是協定，akisfall.github.io的部分就是網域，hexoproject是文件路徑
協定，就像是網路上統一的溝通格式，可以讓Windows、Linux、Mac跨平台顯示。
網域，類似於別稱，如果今天我要連facebook結果我要打31.13.87.36 我相信，誰都不想要登了對吧？但我改成facebook.com，好記又好用。因此網域伺服器（DNS）便由此得生。
路徑，就是在那個網域下面的檔案，hexoproject就是指這個。

財政部的URL

直接觀察財政部的發票兌獎網站我們可以發現。
https://www.etax.nat.gov.tw/etw-main/ETW183W2_11301/
在ETW183W2_，後面有個可以被辨別的文字，也就是民國年加上月份(11301)。
那我們要怎麼確保ETW183W2_都不會變動呢？如果你把11301改成11211或者多試幾個，還能夠找到網頁，就代表財政部是循著這個規則發布發票兌獎的。
通常網站的URL在設計的時候，都會有幾個參數能夠讓人類辨識，即便在多亂碼也是如此。

專案起頭

在專案資料夾內，建立一個python檔，名稱可以自己取，以下用receipt為例。
如果你打開Vscode的話，它會自動偵測虛擬環境，就按藍色的按鍵給他設定就好了。

利用Python內建的datetime，我們可以製造出0X月的格式，就能跟財政部的0X月格式一模一樣。

year

year需要轉型再轉型的原因是，datetime.strftime("%Y")回傳的year是字串，因為要計算民國年，而回傳的是西元年，所以要優先轉型int扣掉1911後，在轉型回字串並加入在url中。

month跟day

先把12個月的所有開獎月跟開那些月份寫出來。

在發票的開獎月中，我們可以知道幾件事情

  • 年初1月開的是去年11、12月的獎。
  • 三月25日以前，因為沒有開獎的原因，所以跟著1月2月爬去年11月、12月的資料。
  • 開獎月開的月份是前2個月，例如：3月開的是1月2月，5月開的是3月4月，所以在第二個elif的結果會是月份減2。
  • 開獎月25號前，只能去抓4個月前的開獎號碼，為甚麼呢？因為開獎開的是前兩個月的發票，最新的資料只有4個月前的資料，所以才會減4。
  • 偶數月不開獎，所以四月爬一月，六月爬三月，八月爬五月、十月爬七月、十二月爬九月的資料。

爬下URL

簡單的靜態爬蟲是透過程式設計來模擬人類在網路上的操作，將想要的資料透過程式擷取（又稱為資料清洗），但爬蟲有不少的限制，因為爬得好可以幫助世人，爬不好就是刑事責任。

robots.txt

robots.txt就是在告訴你，我這個網站什麼能爬，什麼不能爬。
所以撰寫爬蟲時，請注意這個重要的文件喔！如果沒有，也要注意別爬到重要資料！！！
如果要查看robots.txt可以在網域後面直接輸入robots.txt
以Youtube為例，財政部網站沒有

User-agent:* 指的就是你的瀏覽器，通常會是瀏覽器的版本或者簡單的電腦資訊等
Disallow:… 就是指你不能看到什麼，或者爬蟲不能看到甚麼文件裡面的內容。

爬蟲禮儀

因為爬蟲主要還是透過連線的方式，所以千萬千萬不要寫一個無窮迴圈，一直送Requests給對象主機，這是非常惡劣的行為，也是違法的網路攻擊行為。
雖然在那之前，你會被對方的主機封鎖IP最後判別為DOS攻擊。
無論如何，爬蟲是可能根據你的行為犯法的，請不要造成對方的困擾！，也讓自己遠離刑事責任！
了解完禮儀後，我們就開始爬下資料吧！

以上的程式碼如果執行過後，應該會輸出很多令人眼花撩亂的文字，沒關係，因為之後這些文字會被清洗成能夠讓人閱讀的文字。

requests

requests套件是一個提供Python能夠進行網頁四大動作的套件
如GET、POST、PUT以及DELETE。
GET是從網頁下面抓取內容的動作、POST則是將資料傳輸給伺服器的動作、PUT則是修改資料的動作、DELETE是刪除的動作。本次會使用到GET，也就是抓取內容的動作，剩下的有興趣可以 到此 了解。
利用re.get() re.post() re.put() re.delete() 便可以做出相對應的動作。

headers

第二行的headers是甚麼？它的名子就是檔頭，檔頭包含了很多訊息，諸如cookie、使用的Browser（瀏覽器）等等
你可以按F12->網路->按F5重整頁面->隨機點任何一行檔案->擋頭 如圖所示。

而這裡的headers其實不是指這個檔頭而是檔頭最後面的User-agent

用這個User-agent的名義下去爬資料，才比較不會發生爬不到的事情。
所以執行re.get(url,headers=headers)就能夠爬下來了喔！

清洗requests的內容

成功把財政部的網頁內容爬下來之後，我們就來進行資料清洗吧！

BS4

BS4是「美麗的湯」的縮寫，全名為BeautifulSoup。
BS4是很常見的資料清洗工具，主要是解析HTML，使用Parser，也就是第一行root後面的參數'lxml'，然後可以讓設計者快速擷取有興趣的文字。
這裡有個重點，lxml要安裝的！
要使用python -m pip install lxml來安裝喔！
隨後使用的root.select()函數是透過CSS selector的方式來選擇想要的元素
如果對於BS4有更深入的興趣可以到這裡做更深入的了解喔！
爬下來的資料要使用for迴圈的方式來顯示！
如果只單純使用print(td)的話，它只會回傳跟你說這是個OBJ。
因為它的class是bs4.element.tags的關係，所以一定要記得使用迴圈來顯示喔！
所以這也是為甚麼會使用列表生成式來放入資料。

接著再把爬下來的資料放到字典裏面，由上到下的英文分別是。

  • 特別獎
  • 特獎
  • 頭獎*3
    至此，爬蟲的部分就結束了，剩下的指令由Python基礎就可以解決了。

第五步，使用者輸入

最後，我們請使用者輸入發票後三碼，如果輸入字串長度過長的時候，升起一個ValueError跳出程式。
先將後三碼比對（利用字串切片），如果有配對到後三碼，在配對後續的5碼的邏輯下去做。
至此，發票兌獎器全數完成！

總結

這次我們製作了發票兌獎器，有時候直接實作Python的程式語言與套件庫，可以發現到Python的程式格外簡單，相同的程式可能在其他語言實做不到，或者可以但須要千百行以上的程式才有可能達到。
做專案最好玩的點在於你可以看著自己一手打造出一個兌獎器、密碼產生器。
即使其他人做過了，但當你從零開始，想像這個軟體怎麼運作，邏輯怎麼走怎麼做，慢慢摸索慢慢了解就會發現，做專案不是為了創造下個GPT，也不只是為了學習，也是為了考驗自己努力不懈的精神。

此專案可以再多發展的方向。

這專案還有很多美中不足的地方，而這裡就提出來，讓讀者們自行解決。

  • 一定要過了25號後才能夠爬到資料嗎？難道我們不可以直接當天爬到資料嗎？
    • 使用datetime的strftime()可以抓到小時跟分鐘，以及weekday()可以知道今天星期幾
    • 根據開獎時間，或者觀察每次開獎後多久可以看到網頁的時間，來爬取資料。
  • 輸入有點麻煩，每次輸入一張發票就要啟動一次程式，無窮迴圈好像可以一直讓它開著？
  • 輸出很陽春，幾個黑白相間的文字，是不是有其他的輸出方法？
    • 有幾個套件叫做PySide,PyQt,Tkinter 都是製作圖形化介面（GUI），除了美化輸出外，還可以解決第二個問題喔！
    • 有一個套件叫做FastAPI，可以自己設計一個API，將輸出結果傳輸到自己寫的網頁上。

另外的註解

  • 爬蟲除了可以爬發票以外，還可以爬股票資料、天氣資料等，以及可以試著連接API來建立屬於自己的資料系統喔！
  • 最後，這個專案一定還有很多很多需要改正的地方，包含Bugs之類的，多請他人測試或許能夠測出很多Bug喔！
