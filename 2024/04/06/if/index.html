判斷式

發表於 2024-04-06 更新於 2024-04-10 分類於 判斷式

判斷式

在程式語言中，判斷式只是一個比較酷炫的名詞。簡單來說，判斷式就是if（如果）。
就像中文中使用「如果…就…」來組成句子一樣，在程式的世界中，我們也可以做同樣的事情。
舉例來說，「如果小明不喜歡吃披薩，就可能不是義大利人」。

if的程式架構

在if判斷式中，有個東西扮演了一大角色，就是上章運算子提到的條件運算子。條件運算子在if的世界中是不可或缺的存在，只有傳True才能夠執行下面的程式區塊。
程式區塊就是只要正確能夠執行的程式碼，一旦執行結束，程式就結束了。
讓我們來看一個例子：假設我的中文語句是「如果使用者輸入大於2的話，就印出Hello」。

if else

以上的程式碼中，你會發現在你輸入小於2的任何數字（1~所有負數），都不會有任何反應，這是因為你的程式碼沒有為這些數字進行處理，但也不可能一一處理
所以程式的世界中提供了else，來囊括所有的可能性，避免一一處理。
如果將其翻譯成中文，「如果使用者輸入大於2的話，就說Hello，不然就說World」

if elif else

但我們不可能只有兩種判斷的方法，還要再判斷如果使用者輸入等於10呢？
此時，就可以運用到elif了，elif幫助我們去處理程式設計師希望考慮的另外條件。
如果要翻成中文就是「如果使用者輸入大於等於2的話，就說Vincent，如果等於1的時候就說Hello，不然就說World」

有個重要的點，假設今天我們的elif的條件式改成
elif input1 == 10
最後仍然不會執行這個elif喔！主要是因為input1>=2已經先判斷完成為True了，提前跳出程式。

三元判斷式

很多程式語言都有三元判斷式，像是Java、C++、C、JavaScript等等可以簡化if。
但原本Python是不打算加入三元判斷式，主要是不符合Python的設計原則「易讀」以及「易維護」，所以原本的Python是沒有所謂的三元判斷式。
然而，在廣大社群的需求下，Python社群經過了漫長的討論後，加入了三元運算子。
值得注意的是，所有的三元運算子只能簡化if else喔！
